Introducción

Una variable es un contenedor que se usa para almacenar información dentro de ella. En JavaScript tenemos cuatro maneras de declarar una variable, esto es por medio de los comandos var, let, const y no usando nada.

Es importante señalar que las variables no son el valor en si mismo, sino un contenedor de dicho valor.

Crear variables

Para usar una variable primero debe crearse. Para esto, el primer paso se conoce como declarar variables. En JavaScript una variable se declara para indicarle al programa a partir de qué lugar empieza a existir y qué nombre tendrá la misma. Para ello se indica el tipo de dato (var, let, const) seguido del nombre de la variable.

var nombre;
let animales;

El próximo paso es asignarle a la variable un valor, el cual será almacenado dentro de la misma. Esto se conoce como inicializar variables. Para inicializar una variable se usa el signo = seguido del dato que queremos almacenar en la variable.

nombre = "Carlos";
animales = ["perro", "gato", "conejo"];

Una variable puede inicializarse en la misma linea de código que es declarada o también puede iniciarlizarse en una linea posterior. Si un programador trata de leer los datos de una variable que no ha sido inicializada, en JavaScript se le asigna el valor de undefinied.

const pi = 3.1415;
let comida;
console.log(comida); // comida = undefinied
comida = "Manzana";
console.log(comida); // comida = Manzana

NOTA: Las variables declaradas con var y let pueden ser declaradas sin iniciarlizarse y en dicho caso se les asigna el valor de undefinied.

NOTA: Las variables de tipo const deben ser declaradas e inicializadas en la misma línea. No se puede dejar una variable de tipo const sin inicializar.

También podemos crear variables sin indicar su tipo, aunque como pasa con las variables de tipo const, estas deben ser declaradas e inicializadas en la misma linea.

num1 = 2.2;
num2 = 3;
console.log(num1 + num2); // 5.2

Se puede crear más de una variable en la misma declaración si separamos los nombres de las distintas variables con una coma. Incluso es posible inicializarlas selectivamente en caso de usar var o let.

let impresora = "HP", comida = "Pera", numero = "5";

Otro punto a evaluar es la redeclaración de variables. Esto solo es permitido con variables del tipo var, las variables del tipo let y const no pueden ser redeclaradas.

var num1 = 1;
let num2 = 2;
const num3 = 3;

var num1 = 10;
let num2 = 20; // SyntaxError: Identifier 'num2' has already been declared
const num3 = 30 // SyntaxError: Identifier 'num3' has already been declared;

Finalmente, se pueden reasignar nuevos valores a las variables ya creadas con var y let, esto lo hacemos escribiendo el nombre de la variable, seguido de un signo = y finalmente el nuevo valor asignado. Las variables creadas con const no permiten la reasignación de valores.

var num1 = 1;
let num2 = 2;
const num3 = 3;

num1 = 10; // num1 = 10
num2 = 20; // num2 = 20
num3 = 30 // error >> Assignment to constant variable.;

Convención para nombrar variables

El programador es libre de ponerle a la variable el nombre que desee, sin embargo, hay algunas excepciones y reglas básicas a tener en cuenta a la hora de nombrar una variable.

  • Se puede usar caracteres en los rangos [a-z] [A_Z] [0-9], el signo dolar "$" y el guión bajo "_". No se pueden usar otros tipos de caracteres porque causarian errores o serian dificiles de comprender.
  • No usar guión bajo al comienzo del nombre. Esto es usado en algunos constructores en JavaScript y tienen un significado especial, por lo cual crear variables que inicien en guión bajo podria ser confuso.
  • No se puede usar números al comienzo del nombre. Esto no esta permitido y causará errores.
  • Una convención segura para nombrar variables es "lower camel case". Se usa en variables cuyo nombre son dos o más palabras y en ella la primer palabra inicia siempre en minúscula y la primer letra del resto de palabras comienzan en mayúscula. Algunos ejemplos podrian ser nombreCliente, equiposVendidos, colorDeAlerta, etc.
  • Hacer nombre de variables intuitivos que describan el tipo de dato que contiene. No asignar simplmente letras y números o frases largas.
  • Las variables son "case sensitive" (sensibles a mayúsculas/minúsculas), entonces MiEdad es distinto que miEdad.
  • No se pueden usar como nombre de funciones las palabras reservadas por JavaScript, como por ejemplo var, let, const, for, while, etc.

Variables locales y globales

Las variables, sin importar con que instrucción es creada, pueden categorizarse en variables locales y variables globales.

Cuando se inicializa una variable afuera de una función se denomina variable global y se puede acceder y modificar la misma en cualquier parte del programa. Si la variable se crea dentro de una función se la denomina variable local y la misma solo existe y puede ser accedida y editada dentro de la función en la cual es creada.

var dia = "lunes";
function asignarMes(){
var mes = "enero";
console.log(dia); // dia = lunes
console.log(mes); // mes = enero
}
console.log(dia); // dia = lunes
console.log(mes); // error >> mes is not defined

Un detalle a tener en cuenta es que las variables globales se almacenan en la memoria durante toda la ejecución del programa, mientras que las variables locales solo se almacenan en memoria solamente durante la ejecución de la función o bloque. Por esto mismo, lo mas óptimo es intentar trabajar siempre en la medida de lo posible con variables locales para ahorrar recursos.

Otra cosa importante a tener en cuenta es que cuando creamos una variable global y posteriormente creamos una variable local con el mismo nombre, dentro del bloque o función donde fue creada siempre se tiene preferencia sobre la variable local al intentar referenciarla, indistintamente de si la variable es declarada con var, let o const.

const num = 1;
function asignarMes(){
const num = 100;
console.log(num); // num = 100
}
if( true ){
const num = 10000;
console.log(num); // num = 10000
}

Ambitos de variables (scope)

También conocido como alcance de variables o scope de variables, es la zona en la cual las variables creadas tienen alcance, es decir, la zona en la cual se puede acceder a ellas. Según declaremos una variable con var, let o const el comportamiento varía.

Si creamos variables fuera de una función o cualquier otro bloque de código, las mismas serán variables globales y podremos acceder a ellas en cualquier punto del programa, decimos entonces que estas variables tienen un scope global. Entiendase bloque de código a el contenido entre llaves {}, que podria ser una función, ciclo, etc.

En cambio si la variable se crea dentro de una función o un bloque, estas son variables locales y solos existiran dentro de la funcicón o bloque en la cual fueron creadas, decimos entonces que estas variables tienen un scope local. Sin embargo, las variables locales se dividen a su vez en variables con scope de funciones y scope de bloque. Las variables con scope de bloque solo existen dentro del bloque en el cual son creadas, mientras que las variables con scope de funciones solo existen dentro de las funciones en las cuales son creadas.

Las variables locales creadas con var siempre tienen un scope de función, mientras que las variables creadas con let y const siempre tienen un scope de bloque. Esto se ilustra si creamos un bloque (por ejemplo un if) dentro de una función como en el siguiente ejemplo:

function contarTres(){
if( true ){
var varNum = 1;
let letNum = 2;
const constNum = 3;
}
console.log(varNum); // varNum = 1
console.log(letNum); // error >> letNum is not defined
console.log(constNum); // error >> constNum is not defined
}
console.log(varNum); // error >> varNum is not defined

En el ejemplo podemos ver una función contarTres() cuyo contenido es un condicional if y en el contenido del if creamos las tres variables. Las variables son creadas dentro de un bloque (del if) que a su vez pertenecen a una función. Como las variables let y const tienen scope de bloque, si nos salimos del bloque e intentamos leerlos se nos indicara que las variables no fueron definidas, en cambio, la variable con var como tiene scope de función nos permite acceder a su valor por fuera del bloque if, sin embargo, la variable se pierde si salimos de la función en la cual es definida.

Este comportamiento de las variables declaradas con var se mantiene sin importar la profundidad del bloque en las cuales son creadas.

function contarTres(){
if( true ){
if( true ){
if( true ){
var varNum = 1;
}
console.log(varNum); // varNum = 1
}
console.log(varNum); // varNum = 1
}
console.log(varNum); // varNum = 1
}
console.log(varNum); // error >> varNum is not defined

Solamente no podemos acceder al valor de la variable al salir de la función.

Lo último importante sobre este tema es que si declaramos una variable del tipo var dentro de un bloque que no pertenezca a una función, esta será creada de manera global.

if( true ){
var varNum = 1;
}
if( true ){
console.log(varNum); // varNum = 1
}
function mostrarNumero(){
console.log(varNum); // varNum = 1
}

Este comportamiento se debe a que la zona global del código se considera como un scope de función y en el ejemplo anterior se puede considerar al bloque if como un bloque que se encuentra dentro del scope de función global del programa. Como las variables con var tienen scope de función se acaban por sobreescribir. Este comportamiento resulta peligroso ya que si creamos una variable global con var y dentro de un bloque creamos otra variable con var que tenga el mismo nombre, esta última variable reemplazara el contenido original de la variable global.

var varNum = 1;
if( true ){
var varNum = 100;
}
console.log(varNum); // varNum = 100

Elevación de variables (hoisting)

La elevación de variables o hoisting de variables es un fenómeno que ocurre con las variables creadas con var. Las declaraciones de estas variables son asignadas en memoria durante la fase de compilación que se ejecutan al inicio del programa, por lo cual, las variables de tipo var serán declaradas (pero no inicializadas) dentro del scope al que pertenezcan incluso antes de llegar a la linea donde se crea dicha variable. Esto se ilustra en el siguiente ejemplo.

console.log(varNum); // varNum = undefinied
var varNum = 1;
console.log(varNum); // varNum = 1

Como las variables con var tienen un scope de función, si declaramos una variable dentro de un bloque, esta tendra exactamente el mismo comportamiento y será declarada al inicio, incluso antes de ejecutar el bloque que define la variable.

console.log(varNum); // varNum = undefinied
if( true ){
var varNum = 1;
}
console.log(varNum); // varNum = 1

Si declaramos la variable con var dentro de una función, el scope de función que tienen las variables var no permitirá que sea declarada fuera de dicha función, pero si al comienzo de la misma.

console.log(varNum); // error >> varNum is not defined
function mostrarNumero(){
console.log(varNum); // varNum = undefinied
var varNum = 1;
console.log(varNum); // varNum = 1
}
console.log(varNum); // error >> varNum is not defined

Como se mensiono antes, esta característica es solamente de las variables declaradas con var, si intentamos hacer esto con variables declaradas con let o const recibiremos un mensaje de error.

console.log(letNum); // Cannot access 'letNum' before initialization
console.log(constNum); // Cannot access 'constNum' before initialization
let letNum = 2;
const constNum = 3;

Arrays como variables

En JavaScript es posible asignar arreglos (arrays) en las variables. Para esto podemos usar cualquiera de los tres comandos vistos hasta ahora. Para declarar un arreglo debemos poner el contenido del mismo dentro de corchetes y separando sus variables por coma. El tipo de dato que contiene puede ser cualquier cosa. Las propiedades del scope y el hoisting siguen siendo los mismos que se analizaron previamente.

var varArray = [1, 2, 3];
let letArray = ['1', '2', '3'];
const constArray = [true, false];

Cuando declaremos como variable un array es recomendable evitar hacerlo con var para evitar posibles errores debido a su scope y hoisting. Es por esto que se suelen declarar usando let o const, aunque hay que tener en cuenta una pequeña diferencia entre ambos. A los arreglos declarados con let es posible editarle un valor en particular dentro del mismo, cambiar el arreglo por completo o reasignarle un nuevo tipo de dato sin problema alguno.

let letArray = ['1', '2', '3'];
console.log(letArray); // letArray = ['1', '2', '3']

letArray[0] = undefinied; // Cambiamos el primer valor del arreglo
console.log(letArray); // letArray = [undefinied, '2', '3']

letArray[3] = true; // Agregamos un nuevo elemento en la posición 3 del arreglo
console.log(letArray); // letArray = [undefinied, '2', '3', true]

letArray = [100, 200]; // Agregamos un nuevo arreglo a la variable
console.log(letArray); // letArray = [100, 200]

letArray = 1; // Asignamos otro tipo de dato a la variable
console.log(letArray); // letArray = 1

En cambio, los arreglos creados con const solamente permiten cambiar el contenido dentro del arreglo elemento por elemento, pero no se les puede reasignar un nuevo arreglo u otro tipo de variable.

const constArray = [true, false];
console.log(constArray); // constArray = [true, false]

constArray[0] = undefinied; // Cambiamos el primer valor del arreglo
console.log(constArray); // constArray = [undefinied, false]

constArray[2] = true; // Agregamos un nuevo elemento en la posición 2 del arreglo
console.log(constArray); // constArray = [undefinied, false, true]

constArray = [100, 200]; // error >> Assignment to constant variable

constArray = 100; // error >> Assignment to constant variable

Conclusiones

A lo largo de este apartado hemos visto el comportamiento de las variables declaradas con var, let y const bajo distintas circunstancias, vimos como crear y manipular variables, asi como también las diferencias que presentan en el scope y hoisting.

Se recomienda que en la medida de lo posible que todas las variables que se declaren sean a nivel local para evitar almacenar innecesariamente la información en memoria y a su vez que sean variables del tipo let o mejor aún const cuando sepamos que el valor de dicha variable no cambiara a lo largo del programa. Con esto evitaremos los comportamientos extraños que presentan las variables declaradas con var, como aquellos presentados en las secciones de scope y hoisting, razones por las cuales se recomienda evitar su uso.
